## ⚡ Project Overview

**MiroFish** is a next-generation AI prediction engine based on multi-agent technology. By extracting seed information from the real world (such as breaking news, policy drafts, and financial signals), it automatically constructs a high-fidelity parallel digital world. In this space, thousands of agents with independent personalities, long-term memories, and behavioral logic interact freely and undergo social evolution. You can dynamically inject variables through a "God's eye view" to precisely deduce future trends — **letting the future rehearse in a digital sandbox, helping decisions win after a hundred simulated battles**.

> You just need to: Upload seed materials (data analysis reports or interesting novel stories) and describe your prediction needs in natural language.</br>
> MiroFish will return: A detailed prediction report and a high-fidelity digital world for deep interaction.

### Our Vision

MiroFish is dedicated to building a swarm intelligence mirror that maps reality, breaking the limitations of traditional prediction by capturing group emergence triggered by individual interactions:

- **On a macro level**: We are a rehearsal laboratory for decision-makers, allowing policies and public relations to trial-and-error with zero risk.
- **On a micro level**: We are a creative sandbox for individual users. Whether deducing the ending of a novel or exploring imaginative ideas, it is fun, engaging, and accessible.

From serious predictions to fun simulations, we make every "what if" visible, making predicting anything possible.

## 🔄 Workflow

1. **Graph Construction**: Real-world seed extraction & individual and group memory injection & GraphRAG construction.
2. **Environment Setup**: Entity relationship extraction & persona generation & environment configuration Agent injection simulation parameters.
3. **Start Simulation**: Dual-platform parallel simulation & automatic parsing of prediction needs & dynamic updating of time-series memory.
4. **Report Generation**: ReportAgent possesses a rich toolset to interact deeply with the post-simulation environment.
5. **Deep Interaction**: Converse with anyone in the simulated world & converse with the ReportAgent.

## 🚀 Quick Start

### I. Source Code Deployment (Recommended)

#### Prerequisites

| Tool | Version Requirement | Description | Installation Check |
|------|---------|------|---------|
| **Node.js** | 18+ | Frontend runtime environment, includes npm | `node -v` |
| **Python** | ≥3.11, ≤3.12 | Backend runtime environment | `python --version` |
| **uv** | Latest | Python package manager | `uv --version` |

#### 1. Configure Environment Variables

```bash
# Copy the example configuration file
cp .env.example .env

# Edit the .env file and fill in the necessary API keys
```

**Required Environment Variables:**

```env
# LLM API Configuration (Supports any LLM API with OpenAI SDK format)
# Recommended to use Alibaba Bailian Platform qwen-plus model: https://bailian.console.aliyun.com/
# Note: Consumption is high, it is recommended to try a simulation with fewer than 40 rounds first
LLM_API_KEY=your_api_key
LLM_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
LLM_MODEL_NAME=qwen-plus

# Zep Cloud Configuration
# The monthly free quota is sufficient for simple usage: https://app.getzep.com/
ZEP_API_KEY=your_zep_api_key
```

#### 2. Install Dependencies

```bash
# One-click installation of all dependencies (Root + Frontend + Backend)
npm run setup:all
```

Or install step-by-step:

```bash
# Install Node dependencies (Root + Frontend)
npm run setup

# Install Python dependencies (Backend, automatically creates virtual environment)
npm run setup:backend
```

#### 3. Start Services

```bash
# Start both frontend and backend simultaneously (execute in project root directory)
npm run dev
```

**Service Addresses:**
- Frontend: `http://localhost:3000`
- Backend API: `http://localhost:5001`

**Start Separately:**

```bash
npm run backend   # Start backend only
npm run frontend  # Start frontend only
```

### II. Docker Deployment

```bash
# 1. Configure environment variables (same as source code deployment)
cp .env.example .env

# 2. Pull image and start
docker compose up -d
```

By default, it reads the `.env` in the root directory and maps ports `3000 (Frontend) / 5001 (Backend)`.

> Accelerated mirror addresses are provided via comments in `docker-compose.yml`, replace as needed.
